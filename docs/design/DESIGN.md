# 디자인 시스템

> web(React)·mobile(React Native) **통일 UI**의 단일 출처.
> 원칙: 컴포넌트 코드는 플랫폼별로 다를 수 있어도, **토큰(색·타이포·간격·라운드·모션)은 공유한다.**

## 통일 전략

- **Design Tokens 단일 출처** — 플랫폼 무관 TS/JSON 토큰을 모노레포 공유 패키지(`packages/design-tokens`
  또는 유사 위치)에 두고 web·mobile 이 함께 소비. 토큰 정의는 `tokens.md` 와 코드가 미러.
- 컴포넌트는 **계약(Props·동작) 공유 + 플랫폼별 구현**. (한 단계 더: Tamagui 로 RN+web 통합 가능 — 추후 검토)
- 상태: `components.md` 에 web/mobile 패리티 추적.

## 방향: A — 따뜻함 (Warm) · 2026-06-25 확정

원칙 한 줄: **"개인 자산 곳간 — 따뜻하고 차분하게, 데이터는 정직하게."** 정보 밀도는 높되 과장 없이.

### 색 (라이트)
| 토큰 | 값 | 용도 |
|------|-----|------|
| `bg` | `#FAF7F1` | 페이지 배경 (웜 크림) |
| `surface` | `#FFFFFF` | 카드·헤더 |
| `ink` | `#2A2521` | 본문 텍스트 |
| `muted` | `#8A8175` | 보조 텍스트·라벨 |
| `line` | `#EAE4DA` | 경계선 (웜) |
| `accent` | `#B5703C` | 브랜드 — 로고·주버튼·링크 (clay) |
| `accent-hover` | `#9E6033` | 주버튼 hover |
| `gain` | `#E5494D` | 수익 (빨강 — 한국 관습) |
| `loss` | `#2F6FED` | 손실 (파랑) |

> 다크 모드는 추후. gain/loss 는 브랜드와 무관한 **고정 의미색**.

### 타이포
- 폰트: **Pretendard** (한국어 표준).
- 숫자: `tabular-nums` (금액·% 세로 정렬).
- 스케일: 라벨 12 · 본문 14 · 카드값 18–20 · 타이틀 24.

### 간격·형태
- 베이스 4px. 카드 패딩 16–20, 라운드 12–16.

### 차트
- 시세 라인 = `accent`(clay). 손익색은 `gain`/`loss`. 기준선 = `line`/`muted`.

## 토큰의 위치 (현 단계)
별도 공유 패키지를 만들지 않고, **각 앱 테마가 이 문서를 단일 출처로 미러**한다:
web = `web/src/index.css` `@theme`, mobile = RN 상수(추후), Keycloak = `login.css` 변수(추후).
(공유 npm 패키지화는 필요해지면 `docs/adr/` 에 결정.)
