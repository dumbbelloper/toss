> **제품명: 곳간 (Gotgan)** — "내 재물을 갈무리하는 곳". 자산관리·백테스팅·은퇴 시뮬을 품는다.
> (레포 코드네임·API 제공자는 `toss`(토스증권)이지만, **서비스 브랜드는 곳간**. 토스는 데이터 출처일 뿐.)
> 로고: 창고 지붕 + 갈무리되는 자산(상승 막대), 공유 SVG `web/src/ui/Logo.tsx` · `mobile/src/ui/Logo.tsx`.

> 이 문서는 "무엇을, 누구를 위해, 왜" 를 정의하는 **단일 진실 공급원**이다.
> 실행 방법(how)은 각 폴더 README·`docs/engineering/`, 결정 근거는 `docs/adr/` 참고.

## 한 줄

토스증권 Open API 기반 **개인 자산 슈퍼앱**. 단순 API 연동 서비스가 아니라, 독립적으로 완성되는
기능 페이지("기둥")들을 쌓고, 그 위에 통합 기능을 얹는다.

## 사용자 · 스코프

- **첫 사용자 = 개발자 본인(1인).** 멀티테넌시·타인 데이터는 현재 범위 밖(아래 *Parked* 참고).
- 따라서 지금은 *내 토스 계정의 자산을 잘 보고·분석·계획*하는 데 집중한다.
- 자동매매는 한다면 **알림 + 수동 확인**까지. 자동 집행은 최후순위(규제·금전 리스크).

## 제품 원칙

1. **예측을 단정으로 팔지 않는다.** 백테스트·시나리오·정보 정리로. 점추정 대신 분포/fan chart.
2. **실존하는 데이터만.** 라이선스를 준수한다(`docs/data/sources.md`).
3. **세금·비용을 빼먹지 않는다.** 세후·건보료·수수료까지 반영해야 "현금흐름"이 진짜가 된다.
4. **각 페이지는 혼자서도 쓸모 있어야 한다.** 통합은 보너스지 전제가 아니다.

## Parked — 지금은 하지 않음 (기록만 남김)

과거 아이디어 중 현 스코프에서 제외한 것. 폐기가 아니라 **보류**.

- **멀티테넌시 + BYOK** (Jira/Confluence식 개인 공간, 타인 toss/telegram 키 저장)
  → 1인 스코프엔 불필요. 타인 증권 API 키 저장은 보안·법적 책임이 큼. 사용자 확장 시 재검토.
- **뉴스·역사 학습으로 금/유가/주식 "예측"**
  → 시장 방향 예측은 비현실적. 재구성하면 가치 있음: 뉴스→이벤트 타임라인+감성 태깅(설명용),
    백테스트 기반 what-if. (예측이 아니라 정보 정리)
- **보유 주식 n년 후 수익 점추정**
  → 단정형 점추정은 오도. 몬테카를로 **분포/fan chart** 로만 제공.

## 부록 — 원본 아이디어 메모 (보존)

> 2026-06 루트 README 에 남긴 원문. 위 *Parked* 의 출처.

- jira, confluence 처럼 SPA 이면서 로그인 시 로그인한 유저의 공간만 보이도록 멀티테넌시를 지원하는 방향으로 발전 가능한가?
  - 로그인 후 개인의 toss, telegram key를 설정한 경우, 해당 키로 개개인의 데이터를 가져와서 차트를 그려주고, 알림을 보내주고, 경우에 따라 매수/매도 자동화까지.
- portfoliovisualizer 처럼 관심 종목들의 백데이터를 전부 저장하고, 당시의 역사들(정치·금리·전쟁 등)까지 학습. 실시간 크롤링/웹검색으로 주요 언론 기사를 학습하여 금·은·유가·주식 전망을 예측.
- 현재 보유 중인 주식들이 n년 뒤 어느 정도 수익을 낼 수 있는지 예측.
